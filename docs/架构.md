# fuelConsumption 项目架构

## 项目定位

fuelConsumption 是一个离线优先的 Flutter 能耗记录应用，用于管理多辆车的加油、充电、插混补能和维修费用记录。当前实现重点是本地记录、能耗/费用统计、趋势展示，以及通过 JSON 手动导入导出备份数据。

应用目前以 Android 验证为主；Flutter 默认生成的其他平台目录仍在工作区中，但本仓库当前通过 `.gitignore` 不纳入版本控制。核心运行数据存储在本机 SQLite 数据库中，不依赖远程服务。

## 技术栈

- Flutter / Dart：应用框架和语言。
- Material 3：主要 UI 组件体系。
- flutter_riverpod：当前用于根部 `ProviderScope`，业务对象仍以显式构造参数注入为主。
- drift + sqlite3_flutter_libs：本地 SQLite 持久化和类型安全查询。
- path_provider + path：定位应用文档目录并创建数据库文件路径。
- fl_chart：仪表盘趋势图。
- collection：领域统计中的集合排序辅助。
- uuid：车辆和记录 ID 生成。
- build_runner + drift_dev：生成 Drift 数据库代码。

## 分层职责

### 入口层

`lib/main.dart` 负责 Flutter 初始化，并组装真实运行依赖：

- 创建 `AppDatabase`。
- 创建 `FuelRepository`。
- 将 repository 注入 `FuelConsumptionApp`。

### UI 层

`lib/src/app.dart` 是当前主要界面文件，负责页面编排、车辆选择、补能类型选择和导入导出调度。边界清晰的表单和展示组件已经拆到 `lib/src/widgets/`：

- `FuelConsumptionApp`：配置主题和根页面。
- `DashboardScreen`：车辆选择、记录列表、统计卡片、趋势图、导入导出入口。
- `VehicleDialog`：车辆新增底部表单。
- `RecordDialog`：补能记录底部表单，支持加油、充电和插混；燃油记录支持优惠金额和实付金额。
- `MaintenanceDialog`：维修记录底部表单。
- `TextPayloadDialog` / `ImportDialog`：JSON 备份展示和导入弹窗。
- `VehicleHeader`、`RecentRecords`、`DashboardMetricGrid`、`TrendCard` 等展示组件。

当前已拆出的组件包括备份弹窗、车辆表单、补能表单、维修表单、车辆概览头、最近记录、统计指标网格、趋势图、能源视觉 helper 和区块标题。`app.dart` 仍承担页面级编排和补能类型选择职责，后续可以继续按“页面、调度、展示组件”拆分。

### 领域层

`lib/src/domain/models.dart` 定义核心业务对象：

- `VehicleType`：燃油车、电车、插混、摩托车。
- `EnergyType`：加油、充电、油电。
- `ChargeMode`：慢充、快充。
- `Vehicle`：车辆基础信息。
- `EnergyRecord`：补能记录，覆盖燃油、充电和插混三种构造方式；燃油记录可保存折前单价、折前金额、折扣、实付金额和折后单价。
- `MaintenanceRecord`：维修/保养等非能源费用记录。
- `StatisticsSnapshot`：UI 展示用统计结果。
- `BackupData`：导入导出的备份数据包。

`lib/src/domain/statistics.dart` 提供 `EnergyStatisticsCalculator`，按车辆类型计算：

- 平均能耗。
- 最近一次能耗。
- 总费用。
- 每公里成本。
- 累计里程。

燃油车和摩托车使用两次加满之间的里程计算油耗；电车按 kWh/100km 计算；插混同时展示油耗和电耗。维修费用可以作为 `maintenanceRecords` 传入统计计算，只影响总费用和每公里成本，不改变能耗标签。

`lib/src/domain/validation.dart` 提供 `RecordValidator` 和 `MaintenanceRecordValidator`，负责：

- 能源数量不能为负。
- 金额和单价不能为负。
- 燃油折扣不能为负，且不能大于折前金额。
- 同一车辆的记录里程必须随日期递增。
- 维修里程和维修费用不能为负。

### 数据层

`lib/src/data/app_repository.dart` 定义 UI 依赖的数据访问抽象：

- 监听车辆列表。
- 监听指定车辆记录。
- 监听指定车辆维修记录。
- 查询车辆、补能记录和维修记录。
- 保存车辆、补能记录和维修记录。
- 导出和导入备份。

`lib/src/data/fuel_repository.dart` 是抽象接口的当前实现：

- 将读写委托给 `AppDatabase`。
- 导出时组装 `BackupData`。
- 导入前校验车辆引用和记录合法性。
- 校验通过后全量替换数据库内容。

`lib/src/data/app_database.dart` 定义 Drift 表和数据库操作：

- `VehicleRows`：车辆表。
- `EnergyRecordRows`：补能记录表。
- `MaintenanceRecordRows`：维修记录表。
- `watchVehicles()`：按车辆名称监听车辆列表。
- `watchRecords(vehicleId)`：按日期倒序监听某辆车记录。
- `watchMaintenanceRecords(vehicleId)`：按日期倒序监听某辆车维修记录。
- `getVehicles()` / `getRecords()` / `getMaintenanceRecords()`：一次性读取。
- `upsertVehicle()` / `upsertRecord()` / `upsertMaintenanceRecord()`：写入或更新。
- `replaceAll()`：导入备份时清空并重建数据。

数据库文件位于应用文档目录，文件名为 `fuel_consumption.sqlite`。`AppDatabase.inMemory()` 可用于测试或临时运行。

数据库 schema version 当前为 `2`。v2 在补能记录表上增加燃油折扣字段，并新增维修记录表；从 v1 升级时通过 Drift migration 追加 nullable 列和新表，避免破坏旧数据。

`lib/src/data/app_database.g.dart` 是 Drift 生成文件，不应手动修改。修改表结构后应运行：

```powershell
dart run build_runner build --delete-conflicting-outputs
```

`lib/src/data/backup_codec.dart` 负责 JSON 备份格式：

- 当前 schema version 为 `1`。
- `encode()` 输出格式化 JSON。
- `decode()` 校验版本并还原车辆、补能记录和维修记录。
- 旧备份缺少 `maintenanceRecords` 时会按空列表处理。

## 运行时数据流

```text
main.dart
  -> FuelConsumptionApp(repository)
  -> DashboardScreen
  -> AppRepository
  -> FuelRepository
  -> AppDatabase
  -> SQLite
```

展示数据流：

1. `DashboardScreen` 调用 `repository.watchVehicles()` 监听车辆列表。
2. 用户选中车辆后，界面调用 `repository.watchRecords(vehicle.id)` 监听该车补能记录，并调用 `repository.watchMaintenanceRecords(vehicle.id)` 监听维修记录。
3. UI 将车辆、按时间排序的补能记录和维修记录传给 `EnergyStatisticsCalculator.build()`。
4. `StatisticsSnapshot` 被渲染为概览指标、趋势图和记录列表。

写入数据流：

1. 用户通过底部表单创建车辆、补能记录或维修记录。
2. UI 构造 `Vehicle`、`EnergyRecord` 或 `MaintenanceRecord`。
3. 补能记录先经过 `RecordValidator` 校验，维修记录经过 `MaintenanceRecordValidator` 校验。
4. UI 调用 `repository.saveVehicle()`、`repository.saveRecord()` 或 `repository.saveMaintenanceRecord()`。
5. `FuelRepository` 委托 `AppDatabase` 写入 Drift/SQLite。
6. Drift stream 推送变化，界面自动刷新。

备份数据流：

1. 导出时，`FuelRepository.exportBackup()` 读取车辆、补能记录和维修记录。
2. `BackupCodec.encode()` 将 `BackupData` 转为 JSON。
3. 导入时，`BackupCodec.decode()` 解析 JSON。
4. `FuelRepository.importBackup()` 校验引用完整性和记录顺序。
5. `AppDatabase.replaceAll()` 全量替换本地数据。
