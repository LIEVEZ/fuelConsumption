# fuelConsumption 项目架构

## 项目定位

fuelConsumption 是一个离线优先的 Flutter 能耗与费用记录应用，用于管理多辆车的加油、充电、插混补能和保养费用记录。当前实现重点是本地记录、油耗/费用统计、趋势展示、车辆管理，以及通过 JSON 手动导入导出备份数据。

应用目前以 Android 验证为主。核心运行数据存储在本机 SQLite 数据库中，不依赖远程服务。

## 技术栈

- Flutter / Dart：应用框架和语言。
- Material 3：主要 UI 组件体系。
- flutter_riverpod：托管 repository 生命周期，并保留后续迁移页面状态和 stream 的空间。
- drift + sqlite3_flutter_libs：本地 SQLite 持久化和类型安全查询。
- path_provider + path：定位应用文档目录并创建数据库文件路径。
- fl_chart：仪表盘趋势图。
- collection：领域统计中的集合排序辅助。
- uuid：车辆和记录 ID 生成。
- build_runner + drift_dev：生成 Drift 数据库代码。

## 分层职责

### 入口层

`lib/main.dart` 只负责 Flutter 初始化并启动 `FuelConsumptionApp`。真实数据库和 repository 由 `repositoryProvider` 创建并在 provider dispose 时关闭数据库连接。

### UI 层

- `FuelConsumptionApp`：配置 `ProviderScope`、主题和根页面。
- `DashboardScreen`：负责页面脚手架、弹窗展示和 repository 调度。
- `DashboardController`：管理底部 tab、当前页面、车辆选择和记一笔入口的 UI 状态转换。
- `widgets/dashboard/*`：Dashboard 加载和错误等纯展示状态。
- `ConsumptionScreen`：油耗首页编排，车辆状态、首页费用、统计卡片和优惠横幅由 `widgets/consumption/*` 承担。
- `ExpenseScreen`：费用页编排，概览、指标、构成、年度统计和明细由 `widgets/expense/*` 承担。
- `ConsumptionTrendCard` / `MonthlyFuelCostCard` / `AnnualConsumptionCard`：首页图表组件。
- `AnnualExpenseCard` / `ExpenseDetailCard`：费用页年度图表和明细组件。
- `RefuelScreen`：优惠加油表单页面，负责日期、选项、保存调度和错误展示。
- `RefuelFormController`：管理加油页文本控制器和机显金额、优惠、实付金额联动。
- `widgets/refuel/*`：加油页日期、里程、金额、分段开关和选填项表单组件。
- `MaintenanceScreen`：保养/维修等非能源费用表单。
- `MineScreen`：车辆管理、本地备份、帮助、反馈和隐私说明。
- `widgets/mine/*`：我的中心的分组卡片、菜单项、车辆行、说明弹层和反馈弹层。
- `CreateRecordSheet`：底部“记一笔”选择加油或保养。
- `VehicleDialog`、`ImportDialog`、`TextPayloadDialog`：车辆输入、备份导入交互和 JSON 展示弹窗；车辆保存和导入执行通过 callback/action 注入。

当前 `ConsumptionScreen` 和 `ExpenseScreen` 已收敛为页面编排，主要卡片迁移到 `widgets/consumption/*` 和 `widgets/expense/*`。`RefuelScreen` 的金额联动和文本控制器已迁移到 `RefuelFormController`，页面仍负责保存流程和系统弹层。

### 应用层

- `BackupImportService`：负责 JSON 解析、备份语义校验、导入前自动备份和导入执行。
- `dashboardQueryProvider`：组合车辆、补能记录和保养记录 stream，输出 `DashboardLoadState`。
- `DashboardData`：封装 Dashboard 页面所需的选中车辆、记录排序、保养记录和统计快照。
- `VehicleCommandService`：负责把车辆表单 draft 转换为带 ID 的 `Vehicle` 并调用 repository 保存。
- `ImportDialogActions` / `VehicleDraft`：作为 UI 和应用服务之间的轻量边界，避免 widget 直接依赖 repository、codec 或 UUID 生成。

### 领域层

`lib/src/domain/models.dart` 定义核心业务对象：

- `VehicleType`：燃油车、电车、插混、摩托车。
- `EnergyType`：加油、充电、油电。
- `ChargeMode`：慢充、快充。
- `Vehicle`：车辆基础信息。
- `EnergyRecord`：补能记录，覆盖燃油、充电和插混。
- `MaintenanceRecord`：保养/维修等非能源费用记录。
- `StatisticsSnapshot`：UI 展示用统计结果。
- `BackupData`：导入导出的备份数据包。

领域计算分布如下：

- `statistics.dart`：计算平均能耗、最近能耗、总费用、每公里成本和累计里程。
- `consumption_statistics.dart`：首页费用与优惠统计。
- `expense_statistics.dart`：费用页汇总、年度聚合和明细列表。
- `fuel_grades.dart`：燃油标号常量。
- `record_amounts.dart`：优先读取补能记录中的结构化机显金额、优惠和实付金额，并兼容旧备注解析。
- `refuel_amount_calculator.dart`：加油页机显金额、优惠和实付金额联动计算。
- `refuel_record_assembler.dart`：将加油表单草稿组装为 `EnergyRecord`，并返回表单级错误。
- `validation.dart`：补能记录和保养记录校验。

### 数据层

`AppRepository` 定义 UI 依赖的数据访问抽象，覆盖车辆、补能记录、保养记录、备份导入导出和备份校验。

`FuelRepository` 是当前实现：

- 将读写委托给 `AppDatabase`。
- 保存车辆、补能记录和保养记录前执行领域校验。
- 导出时组装 `BackupData`。
- 导入前通过 `BackupValidator` 校验 ID 唯一性、车辆引用和记录合法性。
- 校验通过后全量替换数据库内容。

`AppDatabase` 暴露数据库读写 API，具体职责拆分为：

- `app_database_tables.dart`：`VehicleRows`、`EnergyRecordRows`、`MaintenanceRecordRows` 表定义。
- `app_database_migrations.dart`：schema v2/v3 迁移和 foreign key 开启。
- `app_database_mappers.dart`：row/domain companion 和实体映射。
- `app_database_connection.dart`：真实 SQLite 连接和内存测试连接。
- `app_database_vehicle_dao.dart`：车辆写入、删除和车辆列表查询。
- `app_database_record_dao.dart`：补能记录写入和查询。
- `app_database_maintenance_dao.dart`：保养记录写入和查询。
- `app_database_backup_dao.dart`：导入备份时的整库替换事务。

数据库文件位于应用文档目录，文件名为 `fuel_consumption.sqlite`。数据库 schema version 当前为 `4`：v2 追加车型/单价字段，v3 新增保养记录表，v4 新增机显金额、实付金额和优惠金额结构化字段。`app_database.g.dart` 是 Drift 生成文件，不应手动修改。

## 运行时数据流

```text
main.dart
  -> FuelConsumptionApp
  -> repositoryProvider
  -> DashboardScreen
  -> AppRepository
  -> FuelRepository
  -> AppDatabase
  -> SQLite
```

展示数据流：

1. `vehiclesProvider` 监听车辆列表。
2. `dashboardQueryProvider` 解析当前车辆后监听该车补能记录和保养记录。
3. `DashboardLoadState` 表示 loading、empty、ready、error，并保留可用车辆上下文。
4. `DashboardScreen` 根据 `DashboardLoadState` 渲染油耗、费用、加油、保养或我的中心页面。

写入数据流：

1. 用户通过底部表单创建车辆、补能记录或保养记录。
2. UI 构造 `Vehicle`、`EnergyRecord` 或 `MaintenanceRecord`。
3. `FuelRepository` 执行校验。
4. `AppDatabase` 写入 Drift/SQLite。
5. Drift stream 推送变化，界面自动刷新。

备份数据流：

1. 导出时，`FuelRepository.exportBackup()` 读取车辆、补能记录和保养记录。
2. `BackupCodec.encode()` 将 `BackupData` 转为 JSON。
3. 导入时，`ImportDialog` 将输入交给 `BackupImportService.parseAndValidate()`，拿到可预览的 `BackupData`。
4. 用户确认替换后，`BackupImportService.importBackup()` 先导出现有数据，再调用 `FuelRepository.importBackup()`。
5. `FuelRepository.importBackup()` 使用 `BackupValidator` 再次校验并调用 `AppDatabase.replaceAll()`。
